# Kyneto ML Sidecar - Data Dictionary

This folder contains the datasets used and generated by the Kyneto Machine Learning pipeline. The datasets are versioned with `_v2` to reflect the current high-fidelity simulation and noise-balanced specifications.

---

## 1. `providers_v2.csv`
**Source:** `simulator.py`
**Purpose:** Contains individual provider profiles and cumulative performance metrics. Each row represents a single storage provider.

| Feature | Type | Description |
|:---|:---:|:---|
| `provider_id` | String | Unique Ethereum-like address of the provider. |
| `stake` | Numeric | Amount of KYN tokens staked by the provider (500 - 100,000 range). |
| `capacity_gb` | Integer | Total storage space pledged to the network in Gigabytes. |
| `utilization_gb` | Integer | Current storage used by active deals. |
| `region` | String | Geographic region of the node (e.g., US-East, EU-West). |
| `node_age_days` | Integer | Days elapsed since the provider registered on the network. |
| `uptime_pct` | Float | Percentage of time the node was reachable (0.0 to 1.0). |
| `avg_latency` | Float | Mean response time in milliseconds for protocol interactions. |
| `post_success_rate` | Float | Success ratio of Proof-of-Spacetime challenges (0.0 to 1.0). |
| `proof_misses` | Integer | Total count of failed PoSt challenges. |
| `slashing_count` | Integer | Total number of penalty events (slashing) triggerd by the protocol. |
| `is_failed` | Boolean | **Target Variable:** `True` if the node met failure criteria (misses/slashing/downtime). |
| `registered_at` | Timestamp | ISO-8601 timestamp of registration. |

---

## 2. `events_v2.csv`
**Source:** `simulator.py`
**Purpose:** A time-series log of all protocol interactions. Each row is a single network event.

| Feature | Type | Description |
|:---|:---:|:---|
| `event_id` | String | Unique identifier for the specific event. |
| `provider_id` | String | Reference to the provider involved in the event. |
| `event_type` | String | Type of protocol message: `post_challenge`, `deal_created`, `deal_completed`, `heartbeat`. |
| `timestamp` | Timestamp | Precise ISO-8601 time the event occurred. |
| `success` | Boolean | Whether the interaction was successful or failed. |
| `latency_ms` | Float | Measured response time for this specific interaction. |
| `data_size_gb` | Integer | Size of data involved (0 for non-deal events). |

---

## 3. `training_data_v2.csv`
**Source:** `prepare_data.py`
**Purpose:** The final feature-engineered dataset used for ML model training. Includes one-hot encoded regions and derived ratios.

### Standard Features (Cleaned)
Features from `providers_v2.csv` are cleaned (missing values handled) and normalized.

### Engineered Features (Constructed)
| Feature | Formula / Origin | Why it's useful |
|:---|:---|:---|
| `utilization_ratio` | `utilization / capacity` | Measures load pressure on the node. |
| `stake_per_gb` | `stake / capacity` | Economics: high-stake/low-capacity nodes might be more reliable. |
| `risk_score` | Weighted sum of PoSt misses, Uptime, Latency, and Slashing. | A composite hazard level (0-100). |
| `event_success_rate`| Aggregated from `events_v2.csv` | Real-time success rate over the last 3 months. |
| `event_avg_latency` | Aggregated from `events_v2.csv` | Real-time mean latency over the last 3 months. |

### Encoded Features
| Feature | Transformation | Description |
|:---|:---|:---|
| `region_*` | One-Hot Encoding | Binary flags for each region (e.g., `region_US-East`, `region_Unknown`). |
| `reliability_tier` | Binning (`risk_score`) | Classification target: `platinum`, `gold`, `silver`, `bronze`. |
